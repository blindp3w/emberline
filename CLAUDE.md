# CLAUDE.md

Burn Rate — a procedurally-rendered, offline-capable endless-runner PWA, played in landscape on
iPhone Safari. `README.md` has the full feature/architecture overview, play instructions, and the
live URL; this file is the operating manual for working in the repo.

## Hard constraints (do not violate)

- **Zero dependencies, no build step, no framework.** Everything ships as plain ES modules the
  browser loads directly. Do not add npm runtime/dev dependencies or a bundler. `package.json`
  exists only for `{"type":"module"}` (so `node test.js` runs ESM) and script aliases.
- **No binary assets, ever.** All art is drawn on `<canvas>`; all sound is synthesized with the
  Web Audio API; PWA icons are generated by `scripts/generate-icons.mjs` (a hand-rolled PNG
  encoder over Node's built-in `zlib`). To change visuals/audio/icons, extend the procedural
  generators — never add `.png`/`.jpg`/`.mp3`/etc.

## Commands

- `node test.js` — unit suite (no framework). Must pass before finishing any change.
- `node scripts/generate-icons.mjs` — regenerate `icons/*.png` (also run in CI).
- `python3 -m http.server 8080` — serve locally; the service worker needs http(s), not `file://`.

There is no browser in CI/dev here, so verify game/render code with a Node smoke harness that
stubs DOM/Canvas/Web-Audio, imports `src/game.js`, and steps the loop (see how prior changes were
validated). Pure logic is covered by `node test.js`.

## Architecture — where code goes

- `src/logic.js` — **pure, DOM-free** game rules: collision, speed curve, spawn timing/placement,
  scoring, air control. No `window`/`document`/`canvas`/timers. **All new gameplay logic goes
  here** so it stays unit-testable and is reused unchanged in the browser.
- `src/game.js` — `requestAnimationFrame` loop (delta-time), input, state machine, spawning, glue.
- `src/render.js` — all canvas drawing (procedural sky/grid/parallax/particles, glow, shake).
- `src/audio.js` — Web Audio synth voices + mute, routed through one master gain.
- `test.js` — Node test runner; add assertions for every new `logic.js` helper.

Tuning constants (velocities, gaps, spawn cadence, palette timings) live in the `CONFIG` object in
`src/logic.js`. Adjust them there, not as magic numbers in `game.js`/`render.js`.

## Conventions that bite if ignored

- **Bump `CACHE_VERSION` in `sw.js` on ANY shipped asset change.** The service worker is
  cache-first, so without a bump returning users keep serving stale JS/CSS. Format `burnrate-vN`;
  increment N.
- **All asset paths are relative (`./…`).** The site is served from a GitHub Pages subpath
  (`/burnrate/`); absolute `/…` paths break there.
- **Coordinate model:** logic uses a virtual `WORLD` (1280×720, `groundY` 560) where y grows
  **downward**. The renderer maps WORLD→screen via `view.scale = cssHeight / WORLD.height`. The
  runner is fixed at `worldX`; obstacles and motes scroll left.
- **Input:** touch via `touchstart`/`touchmove`/`touchend`/`touchcancel` (never mouse events),
  with two selectable schemes (toggle persisted to `burnrate.controls`, **default `zones`**).
  **Zones** (multi-touch, press-to-act): right half = jump / mid-air boost, left half =
  hold-to-slide (sit on the ground, fast-fall in the air); releasing the last duck-zone finger
  stands up, with an overpass safety-latch. **Swipe** (Tier-1 fallback): tap/swipe-up →
  `primaryAction` (jump / boost / stand up), swipe-down → `downAction` (sit / fast-fall). Keyboard
  mirrors the swipe mapping and ignores `e.repeat`.

## Player/state model

- Crouch is a **held `sitting`** state (not timed). It auto-stands once a ducked-under OVERPASS has
  passed, but a manual sit with nothing nearby persists until the player stands. In zones mode the
  held duck zone drives it (`sitHeld` gates the auto-stand so a hold persists through obstacles).
- Sitting clears OVERPASS cables but NOT low BARRIERs (those require a jump).
- One mid-air boost per jump, gated by `airBoostUsed` (reset on each ground jump).

## Workflow & deploy

- Keep `node test.js` green; prefer adding a pure helper + tests over inlining logic in `game.js`.
- Pushing to `main` triggers `.github/workflows/pages.yml` (generate icons → run tests → deploy to
  GitHub Pages). Don't break that pipeline; remember the `CACHE_VERSION` bump so the deploy is
  actually picked up by clients.
- Match existing style: plain functions, comments only for non-obvious constraints, and the
  dusk-synthwave palette constants already defined in `render.js` and `styles.css`.
